The `Aimbot` library gives you full control over the aimbot — FOV, smoothing, target selection, keybinds, prediction, easing curves, and more. Everything can be changed on the fly, so you can build dynamic configs that adapt to what's happening in-game.

---

## Toggle

<ResponseField name="Aimbot.Enable()" type="void">
  Turn on the aimbot.
</ResponseField>

<ResponseField name="Aimbot.Disable()" type="void">
  Turn off the aimbot.
</ResponseField>

<ResponseField name="Aimbot.Toggle()" type="boolean">
  Flip the aimbot on or off. Returns the new state.
</ResponseField>

<ResponseField name="Aimbot.IsEnabled()" type="boolean">
  Check if the aimbot is currently active.
</ResponseField>

```lua
if not Aimbot.IsEnabled() then
    Aimbot.Enable()
    notify("Aimbot turned on", "Script", 2, 1)
end
```

## FOV

The FOV circle determines how close a target needs to be to your crosshair before the aimbot kicks in.

<ResponseField name="Aimbot.SetFOV(value)" type="void">
  Set the aimbot FOV radius in pixels.
  <ParamField path="value" type="number" required>
    FOV radius (e.g. `150`). Bigger = more aggressive.
  </ParamField>
</ResponseField>

<ResponseField name="Aimbot.GetFOV()" type="number">
  Returns the current FOV radius.
</ResponseField>

---

## Smoothing

Smoothing makes the aim movement gradual instead of snapping instantly. Higher values = slower, more natural-looking aim.

<ResponseField name="Aimbot.SetSmoothing(x, y?)" type="void">
  Set the aim smoothing factor. If you only pass one value, it's used for both axes.
  <ParamField path="x" type="number" required>
    Horizontal smoothing (e.g. `5.0`)
  </ParamField>
  <ParamField path="y" type="number">
    Vertical smoothing. Defaults to `x` if you don't pass it.
  </ParamField>
</ResponseField>

<ResponseField name="Aimbot.GetSmoothing()" type="number, number">
  Returns the current smoothing as two values: `x, y`.
</ResponseField>

```lua
Aimbot.SetSmoothing(8) -- same smoothing for both axes
local sx, sy = Aimbot.GetSmoothing()
print("Smoothing:", sx, sy)
```

  Choose which body part to aim at.

</ResponseField>

<ResponseField name="Aimbot.GetTarget()" type="number">

</ResponseField>

## Mode

<ResponseField name="Aimbot.SetMode(mode)" type="void">
  Set how the aimbot actually moves your aim.
  <ParamField path="mode" type="number" required>
    | Value | Mode | How it works |
    |---|---|---|
    | `0` | Mouse | Moves your mouse cursor |
    | `1` | Memory | Writes aim angles directly to memory |
    | `2` | Camera | Writes camera angles |
  </ParamField>
</ResponseField>

<ResponseField name="Aimbot.GetMode()" type="number">
  Returns the current aimbot mode.
</ResponseField>

## Prediction

Prediction compensates for target movement by leading your aim. Enable it and optionally set the multipliers for each axis.

<ResponseField name="Aimbot.SetPrediction(enabled, predX?, predY?)" type="void">
  Turn prediction on/off and optionally adjust the prediction multipliers.
  <ParamField path="enabled" type="boolean" required />
  <ParamField path="predX" type="number">
    Horizontal prediction multiplier. Only needed if you want to change the value.
  </ParamField>
  <ParamField path="predY" type="number">
    Vertical prediction multiplier. Only needed if you want to change the value.
  </ParamField>
</ResponseField>

```lua
-- Just enable prediction with current values
Aimbot.SetPrediction(true)

-- Enable and set custom multipliers
Aimbot.SetPrediction(true, 0.15, 0.15)
```

## Easing

<ResponseField name="Aimbot.SetEasingStyle(style)" type="void">
  Set the easing curve for aim movement. This changes how the aim speed varies as it moves toward the target.
  <ParamField path="style" type="number" required>
    | Value | Style |
    |---|---|
    | `0` | Linear |
    | `1` | Sine |
    | `2` | Quad |
    | `3` | Cubic |
    | `4` | Quart |
    | `5` | Quint |
    | `6` | Expo |
    | `7` | Circ |
    | `8` | Back |
    | `9` | Bounce |
    | `10` | Elastic |
  </ParamField>
</ResponseField>

<Tip>
  `Linear` (0) is the most predictable. `Sine` (1) and `Cubic` (3) feel pretty natural. The wilder ones like `Bounce` and `Elastic` are more for fun than for actual aiming.
</Tip>

## Keybind

<ResponseField name="Aimbot.SetKey(vkCode, mode?)" type="void">
  Set the activation key and how it behaves.
  <ParamField path="vkCode" type="number" required>
    [Windows virtual key code](https://learn.microsoft.com/en-us/windows/win32/inputdev/virtual-key-codes) (e.g. `0x02` for right mouse button)
  </ParamField>
  <ParamField path="mode" type="number" default="0">
    | Value | Mode |
    |---|---|
    | `0` | Hold (active while key is held) |
    | `1` | Toggle (press to flip on/off) |
    | `2` | Always on |
  </ParamField>
</ResponseField>

## Sensitivity

<ResponseField name="Aimbot.SetSensitivity(value)" type="void">
  Set the aim sensitivity multiplier. Lower values make the aimbot less aggressive.
  <ParamField path="value" type="number" required>
    `1.0` = default speed, `0.5` = half speed
  </ParamField>
</ResponseField>

## Full Example — Adaptive Aimbot

```lua
Aimbot.Enable()
Aimbot.SetMode(0)        -- mouse
Aimbot.SetTarget(0)      -- head
Aimbot.SetSmoothing(6)
Aimbot.SetFOV(200)
Aimbot.SetKey(0x02, 0)   -- right mouse, hold mode
Aimbot.SetTeamCheck(true)
Aimbot.SetEasingStyle(3) -- cubic
Aimbot.SetPrediction(true, 0.12, 0.12)

-- Dynamically adjust FOV based on closest enemy distance
while true do
    local closest = Player.GetClosest(true)
    if closest then
        local fov = math.max(50, 300 - closest.Distance * 2)
        Aimbot.SetFOV(fov)
    end
    wait(0.05)
end
```
